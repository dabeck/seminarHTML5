Was ist Drag & Drop?

Drag & Drop – "Ziehen und Ablegen"
ist eine Methode zur Bedienung grafischer Benutzeroberflächen von Rechnern
druch das Bewegen grafischer Elemente mittels eines Zeigers.
Quelle: http://de.wikipedia.org/wiki/Drag_and_Drop
Mit der HTML5 Drag & Drop API lassen sich Drag & Drop -Funktionen ohne die Hilfe externer JavaScript-Bibliotheken wie jQuery oder MooTools umsetzen.

Was ist die Drag & Drop API?

  • Drag & Drop in jedem* Browser
  • ohne Bibliotheken
    • kein jQuery
    • kein MooTools
  • jedes Element beweglich
  • vieles mehr...
  • im Internet Explorer konnte man Text/URLs in die Adresszeile ziehen!
  • Grund für die Aufnahme in HTML5 Safari und Mozilla hatten Implementierung vom Internet Explorer übernommen
  • Standardisierung bestehender Techniken: Qualität hin oder her
  • Ian "Hixie" Hickson ist Autor und Betreuer der Acid2 und Acid3-Tests und der HTML 5-Spezifikation.

geschichtlicher Hintergrund

  • stammt vom Internet Explorer 5
  • missraten
  • Grund für die Aufnahme:
    • Safari und Firefox übernahmen die Techologie
    • Standardisierung bestehender Techniken
Quelle: https://twitter.com/Hixie/status/4075253361
  • Ian Hickson erfand das Attribut draggable um mehr als nur (markierte) Bilder und Text zu ziehen.
  • er schrieb es in die HTML5 spezifikation
  • Zunächst nur im FF mittlerweile in jedem Browser implementiert
  • Drag & Drop API stellt ausgeklügelte Formen des Datentransfers während der Operation bereit was mit JS nicht möglich ist!

Was machte eine Implementierung nötig?

  • mehr Möglichkeiten:
    • Cross - Website
    • Cross - Browser
    • Browser - Betriebsystem
  • ausgeklügeltere Formen des Datentransfers:
    • Welche Art von Element/Daten wird verschoben?
    • Wohin dürfen bestimmte Elemente gezogen werden?
  • für komplexere Anwendungsfälle
  • JavaScript Bibs für einfache Anwendungsfälle immernoch ausreichend
  • Tastenkombinationen bei Drag&Drop

Die neuen Möglichkeiten

  • Webseiten interoperabel
  • Desktop- und Online kompatibel
  • Betriebsystemfunktionen möglich
  • einfache Datei-Uploads
  • keine Bibliotheken
  • Derart simples D&D ist mit HTML5 ungleich schwerer.

Implementierung

mit HTML5

Events im Zusammenhang mit Drag & Drop:

dragstart  //gezogenes Element

drag        //gezogenes Element

dragenter   //Element über dem sich das
            //gezogene Element gerade befindet

dragleave   //vorheriges Ziel-Element

dragover   //aktuelles Ziel-Element

drop       //aktuelles Ziel-Element

dragend     //gezogenes Element 

draggable   //Attribut
  • Beispiel der MooTools implementierung jetzt mit HTML5 ungleich komplizierter
  • Man benötigt:
    • draggable="true" erlaubt es das Element zu ziehen
    • ondragstart: Was soll passieren wenn das Element gezogen wird?
      • dragTransfer.setData() gibt die Daten des gezogenen Elements weiter. (hier die ID)
    • ondragover: gibt an auf welchen elementen ein Element fallen gelassen werden kann. Standardmäßig nicht erlaubt deshalb default unterbinden.
    • onDrop: gibt an was passieren soll wenn das element fallen gelassen wird.

Implementierung

mit HTML5

<div id="dragme" draggable="true" ondragstart="drag(event)"></div>
<div class="dropme" ondrop="drop(event)" ondragover="dropPermitted(event)"></div>

<script>
    function dropPermitted(ev){ 
        ev.preventDefault();
    }

    function drag(ev){
        ev.dataTransfer.setData("Text",ev.target.id);
    }

    function drop(ev){
        ev.preventDefault();
        var data=ev.dataTransfer.getData("Text");
        ev.target.appendChild(document.getElementById(data));
    }
</script>

Mit der HTML5 File API lassen sich clientseitig Dateien einlesen und verarbeiten. Egal ob Text oder Binär, zerteilen + versenden etc.

Was ist die File API?

  • Dateibehandlung
  • Dateiverarbeitung
  • Dateistatus
  • Transferinformationen
  • vieles mehr...
  • sehr primitive dateibehandlung
  • man konnte nur EINE datei in ein input-Element eintragen...
  • ...und diese per Formular an den Server senden
  • es gab KEINE rückmeldung über erfolg, fehlschlag oder fortschritt des Uploads

Wie wurden bisher Dateien verarbeitet?

  • sehr primitiv
  • Datei auswählen
  • Formular absenden
  • fertig
  • keine weiteren Möglichkeiten
  • API unterteilt sich in 3 wesentliche APIs
  • Daneben gibt es: BlobBuilder, DirectoryReader, FileEntry/DirectoryEntry, LocalFileSystem
  • FileLists ermöglichen es mehrere Dateien zu verarbeiten
  • Asynchrones einlesen einer/mehrerer Dateien
  • verarbeiten von Dateien heißt: ID3-Tag auslesen, EXIF-Daten auslesen, ZIP-Datei generieren

Was wurde mit der File API verbessert?

  • FileLists
    • File-Objekte
    • Datei-Eigenschaften
    • Dateien zerteilen
  • FileReader
    • Dateien einlesen
    • Dateien verarbeiten
  • FileWriter
    • Dateien verarbeiten
    • Dateien schreiben
  • FileLists ermöglichen es mehrere Dateien zu verarbeiten
  • Asynchrones einlesen einer/mehrerer Dateien
  • verarbeiten von Dateien heißt: ID3-Tag auslesen, EXIF-Daten auslesen, ZIP-Datei generieren
  • Hier wird nur eine Datei gefählt weil das "multiple"-Attribut fehlt.

FileReader

  • stellt Methoden zum einlesen bereit
  • alle Dateioperationen durchführbar
  • Text oder Binärdateien
  • Erster Parameter ist ein FileObjekt (oder Blob)

Möglichkeiten

FileReader

  • stellt folgende Einlesemethoden bereit:
readAsArrayBuffer(file);          //Binärdaten in Typed Array einlesen

readAsBinaryString(file);   	//Inhalt in Binärstring einlesen

readAsDataURL(file);              //Inhalt als Data-URL einlesen

readAsText(file, [, charset]);		//Inhalt als Text einlesen
  • Folgender ablauf beim Einlesevorgang
  • FileReader ermöglicht es auch Synchron einzulesen wenn man mit WebWorkers arbeitet

Möglichkeiten

FileReader

  • Einlesevorgang:
    • Objekt einlesen
    • Status aktualisieren
    • "result" bereit gestellt

  • Status lässt sich abfragen
  • asynchroner ablauf
  • Was bietet der FileReader?
  • 6 File-events!
  • wichtig sind: progress und load
  • vorsicht es geht hier NUR um lokalen progress/events! dies hat nichts mit upload zu tun

Möglichkeiten

FileReader


Es gibt sechs Events beim FileReader:

loadstart  //wenn der Einleiseprozess startet

progress   //jedes eingelesene Byte oder nach 50ms

abort      //wenn das Einlesen abgebrochen wird

error      //wenn ein Fehler auftritt

load       //wenn der Einleseprozess erfolgreich abeschlossen ist

loadend    //wenn der Einladeprozess beendet ist
